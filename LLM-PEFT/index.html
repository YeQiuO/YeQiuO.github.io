大模型的参数高效微调

发表于 2024-07-12 更新于 2024-07-17 分类于 LLM微调
Parameter-Efficient Fine-Tuning

参数微调方法

为什么需要PEFT？

  • 大语言模型参数越来越大，使在消费级硬件上对模型进行全部参数的微调变得不可行
  • 为每个下游任务独立存储和部署微调模型变得非常昂贵，因为微调模型（调整模型的所有参数）与原始预训练模型中需要反向传播的参数量相同。

因此，近年来研究者们提出了各种各样的参数高效迁移学习方法，即固定住预训练模型的大部分参数，仅调整模型的一小部分参数来达到与全部参数的微调接近的效果。调整的可以是模型自有的参数，也可以是额外加入的一些参数，基于此将微调方法分为 A – additive，S – selective，R – reparametrization-based 三大类。

  • 增加额外参数，如：Prefix Tuning、Prompt Tuning、Adapter Tuning及其变体；
  • 选取一部分参数更新，如：BitFit；
  • 引入重参数化，如：LoRA、AdaLoRA、QLoRA；
  • 混合高效微调，如：MAM Adapter、UniPELT；

BitFit

Adapter Tuning

在 Transformer Layer 的 Self-Attetion+FFN 之后插入一个先降维再升维的 MLP（以及一层残差和Layer Normalization）来学习模型微调的知识，具体分为串行和并行两个版本。

图中，Adapter 即插入的 FF Down + 非线性关系 + FF up。在微调过程中，他们固定原始模型的参数，只微调适配层；无需为每个任务存储完整的模型，只需存储适配参数；只需 3.6% 的参数。

缺陷与不足：需要修改原有模型结构，同时还会增加模型参数量。

MAM Adapter

Prompt Tuning

在模型输入的 Token 序列前添加前缀 prompt token，而这个前缀 prompt token 的 embedding 是由网络学到。相当于仅用 prompt token 的 embedding 去适应下游任务，相比手工设计或挑选 prompt，它是一种 Soft 的 prompt（软提示）。

性能如何？

Soft Prompt Tuning 在模型增大时可以达到接近 fine-tuning 的效果；

离散的 Prompt Tuning（Prompt Design）基本不能达到 fine-tuning 的效果，prompt 长度过短时，效果不佳；

P-Tuning

Prefix Tuning

深化 Promot Tuning 的思想，在模型内加入可学习的 Prefix Prompt Token。

为 𝑙 层的 Transformer Layer 的每层多头注意力的键和值都配置了可学习的 prefix vectors。

通过共享连续提示同时优化多个任务。多任务学习是 P-Tuning v2 的可选功能，可以通过提供多任务 Prefix 来进一步提升性能。

在训练样本量较少的场景下，微调结果优于 FineTuning；

prefix长度在 100~200 时，微调结果最优；

**缺陷不足：**由于模型可接受的最大输入长度有限，随着软提示的参数量增多，实际输入序列的最大长度也会相应减小，影响模型性能。

P-Tuning v2

LoRA

深度网络由大量Dense层构成，这些参数矩阵通常是满秩的。相关工作表明，在适应特定任务时，预训练模型是过度参数化的，hidden_size 实际上存在于一个较低的内在维度上，即高维数据实际是在低维子空间中。

因此，将 $$\Delta\mathbf{W}=\mathbf{BA}$$ 用两个更参数量更小的矩阵 $$\mathbf{B}\in\mathbb{R}^{r\times d}$$ 和 $$\mathbf{A}\in\mathbb{R}^{d\times r}$$ 进行低秩近似。具体而言，冻结预训练的模型权重，并将可训练的秩分解矩阵注入到LLM的每个 Transformer Decoder 层中。

h=(W0+ΔW)x=W0x+BAx\mathbf{h}=(\mathbf{W}_0+\Delta\mathbf{W})\mathbf{x}=\mathbf{W}_0\mathbf{x}+\mathbf{B}\mathbf{A}\mathbf{x} h=(W0​+ΔW)x=W0​x+BAx

其中，矩阵 B 通过高斯函数初始化，矩阵 A 为全零初始化，使得训练开始之前旁路对原模型不造成影响。

优势：

  • 可以将原权重与训练后权重合并，因此在推理时不存在额外的开销；
  • 不增加模型参数，不改变模型结构，减少了训练和推理的成本；

近年来，基于Transformer架构的大语言模型（LLM），在众多自然语言处理（NLP）任务达到了最佳性能表现。目前，构建领域专用LLM的主流策略是：在海量通用数据集上预训练的大语言模型（PLM），并通过针对特定下游任务的微调（Fine-tuning）。相较于直接采用PLM，针对下游任务的微调能显著提升模型性能。然而，随着LLM参数规模急剧增长，在消费级硬件上进行微调变得不切实际。

为应对此挑战，研究者们近年来提出了参数高效微调技术（Parameter-Efficient Fine-Tuning, PEFT），其核心思想在于固定PLM的大部分参数，仅调整一小部分关键参数或引入额外参数，以接近直接微调PLM的效果。这种方法不仅保留了PLM的广泛知识库，还显著降低了计算与存储需求，为在资源受限环境下高效微调LLM开辟了新路径。
